Host a project

After you create a SKY UX project, serve the default single-page application, edit pages and components in the project, and run tests for the project, you can follow the steps in this tutorial to host the default SPA.

The only special requirement to host internal Blackbaud implementations is web hooks in the Git repo to the SKY UX continuous integration layer. For now, this must be set up manually by the SKY UX team. Contact the SKY UX team on the 1bb-skyux-apps Slack channel.

When the builder runs in the continuous integration layer, it stores the static files that it produces (JavaScript, JSON, etc.) in the registry for SKY UX Host to serve them.

The only special requirement to host internal Blackbaud implementations is web hooks in the Git repo to the SKY UX continuous integration layer. For now, this must be set up manually by the SKY UX team. Contact the SKY UX team on the 1bb-skyux-apps Slack channel.

SKY UX Host is the gateway for requests to SKY UX SPAs. During local development, SKY UX Host loads local files. And for production, SKY UX Host loads bundled files from a CDN where SKY UX Builder stores them. For more information, see the SKY UX Host section of the overview.
